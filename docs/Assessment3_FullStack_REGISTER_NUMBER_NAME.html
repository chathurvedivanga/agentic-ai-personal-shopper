1. Project Overview

The project developed for this assessment is an AI-driven shopping assistant called Agentic AI Personal Shopper, presented in the interface as AI Shopping Partner. The objective of the frontend is to provide a clean conversational interface through which a user can ask for product recommendations, compare alternatives, review YouTube-based shopping research, revisit earlier conversations, and continue the same chat later.

The frontend is implemented as a single-page React application created with Vite and styled with Tailwind CSS. Instead of using multiple web pages, the application uses a workspace-style interface similar to ChatGPT, Gemini, and Claude. The frontend communicates with the Flask backend through REST APIs and receives live assistant output using Server-Sent Events (SSE).

2. Frontend Architecture

  • App.jsx: main application state, API integration, session loading, SSE streaming, home/chat switching.
  • PromptComposer.jsx: prompt input box, submit handling, Enter-to-send behavior, stop button, and status text.
  • SessionSidebar.jsx: saved chat list, collapsible side panel, new chat action, rename and delete controls.
  • ChatBubble.jsx: renders assistant and user messages, markdown formatting, loading state, and YouTube source chips.
  • lib/sse.js: parses streamed events coming from the Flask backend.

3. User Interface Design

3.1 Home / Landing View

The landing view contains the centered title AI Shopping Partner, a large prompt composer, and starter prompts. This view acts as the clean entry point for starting a new shopping query.

3.2 Conversation Workspace

After the first prompt, the interface switches to a conversation workspace that displays the active chat title, user and assistant bubbles, streaming output, YouTube links, and a bottom prompt composer for follow-up questions.

3.3 Saved Chats Sidebar

The left sidebar stores earlier sessions and allows the user to create a new chat, revisit a saved chat, rename a chat, or delete a chat. This provides persistent conversation management similar to modern AI assistants.

4. Routing and Navigation

This project does not use React Router. Instead, it uses a state-driven SPA navigation model. This is suitable because the application is centered around one conversational workspace rather than multiple URL-based pages.

  • Selecting New chat resets the interface to a draft conversation.
  • Clicking AI Shopping Partner returns the user to the home state.
  • Selecting a saved session loads that conversation from the backend.
  • The sidebar can collapse on desktop and open as a drawer on mobile.

5. User Interaction Handling

5.1 Prompt Input and Form Handling

  • Enter sends the prompt.
  • Shift + Enter inserts a new line.
  • Empty prompts are blocked.
  • The textarea supports a 2000-character maximum.
  • A live character counter is shown.

5.2 API Request Handling

The frontend sends a POST /api/chat request with the new message, prior chat history, and the active session ID. The response is read as an SSE stream and processed using the event types session, status, sources, chunk, error, and done.

5.3 Real-Time Feedback

  • Status text while the backend is preparing research
  • Streaming assistant text while generation is in progress
  • A stop button to cancel active generation
  • Error banners for network or backend failures
  • Automatic scroll to the latest message

6. Responsive User Interface

  • Fixed desktop sidebar and drawer-style mobile sidebar
  • Collapsible sidebar for more content space
  • Scrollable chat area with persistent composer
  • Controlled content width for readability
  • Adaptive spacing and typography using Tailwind utility classes

7. Component-Based Design and Dynamic Data Rendering

The application uses arrays such as sessions and messages to render the sidebar and conversation dynamically. Assistant source links are rendered only when available, and session metadata is updated after rename, delete, or new assistant replies.

8. API Documentation for Frontend Integration

Endpoint Method Purpose
/api/sessions GET Fetch all saved chat sessions for the sidebar
/api/sessions/<id> GET Load a selected conversation and its messages
/api/sessions/<id> PATCH Rename a chat title
/api/sessions/<id> DELETE Delete a saved chat
/api/chat POST Send a prompt and receive a streamed assistant response

9. Conclusion

The frontend of Agentic AI Personal Shopper was implemented as a modern React single-page application focused on readability, responsiveness, and continuity of user interaction. It combines a clean landing page, a chat-based shopping workspace, persistent saved conversations, and live streaming responses. The project demonstrates component-based design, asynchronous UI updates, form handling, responsive layout design, and practical frontend engineering for an AI-assisted product research system.
